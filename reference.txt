1. useState Hook
================

Purpose
-------
The useState hook is used to create and manage local state inside functional components.
It allows components to store and update dynamic data without reloading the page.

Syntax
------
const [state, setState] = useState(initialValue);

How It Works
------------
- 'state' stores the current value.
- 'setState' updates the value.
- Updating state automatically re-renders the component.

Features Implemented Using useState
-----------------------------------
- Counter functionality
- Form handling
- Input field updates
- Toggle buttons
- Dynamic rendering
- User interaction handling

Example
-------
const [count, setCount] = useState(0);

<button onClick={() => setCount(count + 1)}>
   Increment
</button>

Benefits
--------
- Easy state handling
- Real-time UI updates
- Simplifies interactive applications

Components Using useState
-------------------------
- App.jsx
- Home.jsx
- One.jsx
- Two.jsx
- Parent.jsx
- Users.jsx
- UserDetails.jsx

2. useEffect Hook
=================

Purpose
-------
The useEffect hook is used to perform side effects in React components.

Examples of Side Effects
------------------------
- Fetching API data
- Updating DOM manually
- Timers and intervals
- Running code when component loads
- Listening to changes in state or props

Syntax
------
useEffect(() => {
   // side effect code
}, [dependencies]);

Dependency Array
----------------
1. Empty Array []
   Runs only once after component mount.

2. With Dependencies [value]
   Runs whenever dependency changes.

3. Without Dependency Array
   Runs after every render.

Example
-------
useEffect(() => {
   fetchUsers();
}, []);

Features Implemented
--------------------
- Fetching user data from APIs
- Updating UI after data retrieval
- Lifecycle management

Benefits
--------
- Replaces lifecycle methods from class components
- Cleaner and more organized side-effect handling

Components Using useEffect
--------------------------
- Users.jsx
- UserDetails.jsx

3. useContext Hook
==================

Purpose
-------
The useContext hook is used for global state management and data sharing
between multiple components without passing props manually at every level.

Problem Solved
--------------
Normally, data is passed using props:
Parent -> Child -> SubChild

This becomes difficult in large applications.
useContext avoids this issue (called prop drilling).

How Context API Works
---------------------
1. Create Context
2. Provide Context
3. Consume Context using useContext

Example
-------
const UserContext = createContext();

const value = useContext(UserContext);

Features Implemented
--------------------
- Shared global data
- Accessing common information in nested components

Benefits
--------
- Cleaner code
- Easier state sharing
- Reduces unnecessary prop passing

Components Using useContext
---------------------------
- Blog.jsx
- One.jsx
- SubChild.jsx
- Two.jsx

4. useMemo Hook
===============

Purpose
-------
useMemo is used for performance optimization.

It stores calculated values in memory and prevents recalculating them
unless dependencies change.

Problem Solved
--------------
Expensive calculations can slow down applications.
useMemo avoids unnecessary recalculations.

Syntax
------
const memoizedValue = useMemo(() => {
   return expensiveCalculation(data);
}, [data]);

Benefits
--------
- Faster rendering
- Improved performance
- Prevents unnecessary calculations

Component Using useMemo
-----------------------
- Two.jsx

==============================================================

5. useCallback Hook
===================

Purpose
-------
useCallback is used to memoize functions.

It prevents functions from being recreated on every render.

Syntax
------
const memoizedFunction = useCallback(() => {
   // function logic
}, [dependencies]);

Benefits
--------
- Improves performance
- Prevents unnecessary child component re-renders
- Useful when passing functions as props

Component Using useCallback
---------------------------
- One.jsx
